The OSM Programming Language

By: Anna Clayton, Michael Cohoon, and Sean Dunn

Motivation

  • When?
    • Created Spring 2012 for MSCS 610: Advanced Theory of Programming Languages (duh).
  • Why?
    • Programmers tend to lack a sense of humor when they read or write code. We wanted to force them into a chuckle.
  • Who?
    • Programmers with a sense of humor.
  • What?
    • A general purpose language suitable for writing command line scripts, system administrator tasks, and possibly templating.

Syntactic features - Data Structures

Data types: Objects – OSM is completely object oriented. We also support Number, Boolean, etc. which are built in objects. Class based system. Create a variable, set a variable Type checking, casting, etc
